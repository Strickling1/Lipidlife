LipidLife — Setup Guide

Go from these files to a live app on your patients' phones in under 2 hours

1
Create a free Firebase project
Firebase is Google's free database and login service. It stores your patients' lab results and habits securely in the cloud.

→ Go to console.firebase.google.com

  • Click "Add project" → name it lipidlife
  • Disable Google Analytics (not needed) → Create project
  • Once created, click the Web icon </> to add a web app
  • Name it LipidLife App → Register app
  • Copy the firebaseConfig object shown — you'll need it in Step 3
💡 Firebase free tier (Spark plan) supports up to 50,000 reads/day and 20,000 writes/day — more than enough for hundreds of patients.
2
Enable Authentication and Firestore
In your Firebase project:

Authentication:
  • Left sidebar → Authentication → Get started
  • Enable Email/Password sign-in method → Save

Firestore Database:
  • Left sidebar → Firestore Database → Create database
  • Choose Production mode → select region europe-west1 (closest to UAE)
  • After creation, go to Rules tab and paste this rule:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId} { allow read, write: if request.auth.uid == userId || request.auth.token.email == 'doctor@yourdomain.com'; } match /labResults/{doc} { allow read, write: if request.auth.uid == resource.data.userId || request.auth.token.email == 'doctor@yourdomain.com'; allow create: if request.auth != null; } match /taskLog/{doc} { allow read, write: if request.auth.uid == resource.data.userId || request.auth.token.email == 'doctor@yourdomain.com'; allow create: if request.auth != null; } } }
🔒 Replace doctor@yourdomain.com with your actual email. This rule ensures patients only see their own data, and you see everyone's.
3
Paste your Firebase config into the app files
Open index.html and dashboard.html in any text editor (Notepad, VS Code, TextEdit).

Find this section near the top of each file:

const firebaseConfig = { apiKey: "PASTE_YOUR_API_KEY", authDomain: "PASTE_YOUR_AUTH_DOMAIN", ...
Replace each PASTE_YOUR_... value with the actual values from your Firebase project settings.

Also in dashboard.html, find:
const DOCTOR_EMAIL = "doctor@yourdomain.com";
Replace with your real email address.
4
Create your doctor account in Firebase
→ Firebase Console → Authentication → Users → Add user

Add your email and a strong password. This is the account that unlocks the cardiologist dashboard. Patients will self-register through the app.
5
Deploy to Netlify (free hosting — 10 minutes)
→ Go to netlify.com → Sign up free with Google

  • Click "Add new site" → "Deploy manually"
  • Drag and drop your entire lipidlife folder onto the upload area
  • Netlify gives you a live URL instantly, e.g. random-name.netlify.app
  • Optionally rename it to lipidlife.netlify.app in Site settings

Your app is now live at two URLs:
https://lipidlife.netlify.app → patient app
https://lipidlife.netlify.app/dashboard.html → your cardiologist dashboard
💡 Want a custom domain like lipidlife.ae? Register it at Namecheap (~$10/yr) and connect it in Netlify → Domain settings. Takes 10 minutes.
6
Add the app icon (optional but makes it look real)
You need two PNG icon files in your lipidlife folder:

icon-192.png — 192×192 pixels
icon-512.png — 512×512 pixels

Create a simple heart + "LL" icon using Canva (free) — use a navy background (#0F1F3D) with your logo in amber (#F5A623). Export as PNG at the correct sizes. Drop them in the folder and redeploy to Netlify.
7
Install on patients' phones
Send patients your app URL. Then guide them:

iPhone (Safari):
  • Open the URL in Safari (not Chrome)
  • Tap the Share button (box with arrow) at the bottom
  • Tap "Add to Home Screen" → Add
  • LipidLife icon appears on their home screen like any app

Android (Chrome):
  • Open URL in Chrome
  • Tap the 3-dot menu → "Add to Home screen" → Add
  • Or Chrome may show an automatic "Install app" banner
